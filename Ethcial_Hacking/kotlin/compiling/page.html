Download kotlin compiler into your system, and set path to the executable file.

.kt -> extension for the kotlin file.

kotlinc x.kt -include-runtime -d x.jar -> compile  kotlin file
-d option indicates the output path for generated class files which may be a directory or a .jar file.
-include-runtime option makes the resulting .jar file self-contained and runnable by including kotlin runtime library in it.

kotlinc -help -> to see what options you have available.

java -jar x.jar -> to finally run your compiled file.

JAR -> Java Archive. It is a file format based on the popular zip file format and is used for aggregating many files into one.

If we run kotlin file, kotlinc x.kt -> this creates a .class file
javap -c x.class -> gives how a file is working under the hood

Kotlin comes with its own standard class library, called "Kotlin runtime".
We can use "Joda" library to work with date and time.

? -> operator is called "Safe call operator", it avoids the dreadfull NullPointException error.
