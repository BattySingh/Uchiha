Kotlin is a JVM language, and so the compiler will emit Java bytecode. Because of this, Kotlin code can Java code, and vice-versa.

Packages allow us to split code into namespaces. Any file may being with a package declaration:
package com.packt.myproject
class Foo
fun bar(): String = "bar"

The package name is used to give us the Fully Qualified name (FQN) for a class, object, interface or function. In above example, class Foo  has fully qualified name of com.packt.myproject.Foo, top level function bar has fully qualified name of com.packt.myproject.bar.

To enable classes, objects, interfaces, and functions to be used outside of the declared package we must import the required class, object, interface, functions.
import com.packt.myproject.Foo

Wildcard Import: If we have a bunch of imports from the same package, instead of importing each class, function individually, we can import entire package using * operator.
Wildcard imports are especially useful when a large number of helper functions or constants are defined at top level, and we wish to refer to those without using the classname.
