Kotlin has 2 keywords for declaring variables, val and var.
"var" is a mutable variable, which is, a variable that can be changed to another value by reassigning it.
var name = "kotlin";

or var can be initialized later,
var name:String;
name = "Kotlin";

The keyword "val" is used to declare a read-only variable. This is equivalent to declaring a final variable in Java. A val must be initialized when created, since it cannot be changed later.

Even though Kotlin is a strongly--typed language, we do not always need to declare types explicitly.
The compiler attempts to figure out the type of an expression from the information included in the expression. This mechanism is called type inference.

In Kotlin, everything is an Object.
Boxing is the conversion from a primitive type to a wrapper type that types place whenever an object is required but a primitive is presented.
Two different values that are boxed might not use the same instance, so referential equality is not guaranteed on boxed values.

Built-in number types in Kotlin:
1.) Long -> 64 (width)
2.) Int -> 32
3.) Short -> 16
4.) Byte -> 8
5.) Double -> 64
6.) Float -> 32

To create number literals, val int = 123;
long value requires the suffix L, and a float requires a suffix F. double is used  as the default for floating point numbers, and int for integral numbers. The hexadecimal and binary use prefixes 0x and 0b respectively.

Kotlin does not support automatic widening of numbers, so conversion must be invoked explicitly.
ex; val int = 123;
val long  = int.toLong(); -> to convert int to a long value

val float = 32.34f;
val double = float.toDouble(); -> to covert to double

Function for conversions: toByte(), toShort(), toInt(), toLong(), toFloat(), toDouble(), toChar().

The usual bitwise operators are not supported in Kotlin, like left shift, right shift, these are not in-built operators in Kotlin, but named functions instead but can still be invoked like operators.
inverse is not a binary operator, but a unary operator and so is invoked by dot syntax on a number.

val leftshift = 1 sh 2;
val rightshift = 1 sh 2;
val unsignedRightShift = 1 ushr 2;
val and = 1 and 0x00001111;
val or = 1 or 0x00001111;
val xor = 1 xor 0x00001111;
val inv = 1.inv();

Chars represent a single character. Chars also support escaping for characters: \t, \b, \n, \r, ', ", \\, \$.
All unicode characters can be represented using the unicode number, ex: \u1234. char type is not treated as number as in Java.

Strings: Jus like in Java, Strings are immutable. Strings can be created using double quotes or triple quotes.
Double quotes create an escaped string. In the escaped string, special characters, such as new line, must be escaped. val string = "string with \n new line";
Triple quotes create raw strings, in a raw string no escaping is necessary, and all characters can be included.

In Kotlin, we can create an array using function arrayOf().
val array = arrayOf(1, 2, 3, 4);

We can create an Array from an initial size and a function, which is used to generate each element, val perfectSquares = Array(10, {k -> k * k});

Unlike Java, arrays are not treated as special by the language, and are regular collection classes. Instances of arrays provide an iterator function and a size function, as well as get and set function.
Classes: ByteArray, CharArray, ShortArray, IntArray, LongArray, BooleanArray, FloatArray, DoubleArray. These allows performace critical code to use arrays as effectively as they could.
